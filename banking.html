Mobile Banking App - Case Study

Creating a secure and intuitive experience for personal and small business banking users.

Project Overview

Design a mobile banking app aimed at simplifying daily banking activities such as checking balances, money transfers, and bill payments, while ensuring security and accessibility for all user types.

Problem Statement

Users find traditional banking apps difficult to navigate, with key features hidden under complex menus, leading to user frustration and poor task completion rates.

Objective

Create a simplified and secure banking app that allows users to quickly access key features like transfers, bill payments, and transaction history, with an intuitive user interface.

Target Users

  • Everyday users who manage personal finances like checking balances and paying bills.
  • Small business owners requiring fast access to multiple accounts and quick money transfers.

Process

  • Research: Surveyed 7 users; 65% mentioned slow transfer processes; 55% wanted better account overview dashboards.
  • User Personas: John (42), business owner managing multiple accounts; Lisa (30), a casual user paying monthly bills.
  • User Flow: Login > Dashboard > Quick Actions (Transfer Money / Pay Bills / View Transactions).
  • Wireframes: Dashboard with account summary upfront and quick action shortcuts.
  • Visual Design: Deep blue and white color palette for trust and readability, large buttons, and optional biometric login.
  • Prototype: Clickable Figma prototype showing fast flows for key tasks like sending money and viewing balances.

Challenges

Balancing advanced features (e.g., spending analytics) while keeping the core interface simple for casual users.

Key Learnings

Reducing cognitive load and providing familiar navigation patterns helped users complete tasks faster. Trust signals like Face ID and clear balance overviews build user confidence.
